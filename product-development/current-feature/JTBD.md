# Jobs to be Done (JTBD) Analysis

## Primary Jobs

### Job 1: Find Specific Requirements

> "When I'm designing a building, I want to quickly find the specific code requirements so that I can ensure compliance."

**Context:**
- 설계 검토 단계
- 특정 요구사항 확인 필요 (예: Fire Separation, Egress Width)

**Current Solution:**
- PDF 검색 (부정확)
- 목차 수동 탐색 (느림)
- 동료에게 문의 (의존적)

**Pain Points:**
- PDF 검색 결과가 너무 많거나 관련 없음
- 올바른 섹션 찾는 데 5-10분 소요
- 테이블 데이터 복사 어려움

**Desired Outcome:**
- 30초 내 관련 조항 찾기
- 정확한 텍스트 참조
- 테이블 데이터 쉽게 활용

---

### Job 2: Navigate Code Hierarchy

> "When I find a relevant article, I want to see related sections so that I understand the full context."

**Context:**
- 특정 조항 발견 후
- 상위/하위 요구사항 파악 필요

**Current Solution:**
- PDF에서 앞뒤 페이지 탐색
- 목차로 돌아가서 구조 파악

**Pain Points:**
- 코드 구조가 복잡하고 깊음
- 관련 조항 간 연결 파악 어려움
- 여러 창 열어서 비교해야 함

**Desired Outcome:**
- 한 번에 계층 구조 파악
- 상위/하위/형제 조항 쉽게 탐색
- 관련 테이블 자동 연결

---

### Job 3: Compare Codes

> "When working on projects in different provinces, I want to compare code requirements so that I can identify differences."

**Context:**
- 여러 주에서 프로젝트 진행
- National vs Provincial 차이 확인

**Current Solution:**
- 두 PDF 나란히 열기
- 동일 섹션 수동으로 찾아 비교

**Pain Points:**
- 섹션 번호 체계가 다를 수 있음
- 수동 비교는 시간 소모적
- 미묘한 차이 놓칠 수 있음

**Desired Outcome:**
- 동일 주제의 조항 자동 매칭
- 차이점 명확히 표시
- 한 눈에 비교 가능
